          How To Use

          1. Fill in your item's current affixes. Check GA for any Greater Affixes and Enchanted for the affix modified by the Enchantress (at most one).
          2. Fill in up to 4 target affixes — the affixes you want on the finished item. Check Needs Improvement for affixes already present but at an unsatisfactory roll value.
          3. GA preservation is automatic: any GA on a current affix that aligns with a target affix is protected throughout optimization. The optimizer will never recommend an action that risks losing it.
          4. Check Legendary if your item cannot have affixes removed. The Enchantress Available toggle tracks whether the one-time enchant has been used — it resets automatically when an enchanted affix is removed.
          5. Set Gear Slot to your item type to restrict suggestions to slot-legal affixes and reduce compute time. Leave it at Any to allow all affixes.
          6. Set Class to your character class to narrow the Adept skill pool to that class's skills (plus class-agnostic generals). Leave it at Any to keep every class's skills in the pool — useful for browsing but not realistic for actual crafting.
          7. Click Optimize. The recommended cube action appears with success probability and expected steps to a terminal outcome (success or GA loss).
          8. Perform the action in-game, then record it with Apply Last Outcome: pick the modification type and prism used (pre-filled from the recommendation), select the source affix that changed, and type the new affix that appeared. Click Apply to update state, then Optimize again.
          9. Click Reset to clear all state and start over.
          Current Limitations
          • Affix pools are based on prism categories. Slot-accurate roll weights and per-affix frequencies are not modeled, so some suggestions may be theoretically possible but practically rare.
          • Affix values and thresholds are not modeled — the optimizer targets affix identity only, not roll quality (e.g. it cannot require at least 1500 Maximum Life).
          • Gold and crafting material costs are not included in the objective. The optimizer minimizes expected cube steps only.
          • Expected Cube Steps counts steps until a terminal outcome — either success (all targets met with GAs preserved) or failure (a protected GA is lost). A lower step count alongside a lower success probability is not a contradiction: most runs end quickly via failure.
          • Residual cases that exceed the abstract-state or iteration budget return an approximate best-so-far result with explicit diagnostics. Increase Thinking Time to expand the budget; 0 uses the largest configured budget.
